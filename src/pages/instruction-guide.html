üîß Troubleshooting & Common Issues

Issue 1: localStorage Not Persisting

Problem: Cart items disappear when navigating between pages.
Cause: Using file:// protocol instead of HTTP server.
Solution: Deploy to XAMPP server and access via http://localhost/

Issue 2: Cart Badge Not Updating

Problem: Cart badge shows incorrect count or doesn't update.
Cause: updateCartBadge() not called after cart operations.
Solution: Call updateCartBadge() in addToCart(), removeItem(), and on page load.

Issue 3: Category Filter Not Working

Problem: Products don't filter when clicking category buttons.
Cause: Missing data-category attributes on product cards.
Solution: Add data-category="CategoryName" to each product card div.

Issue 4: Stock Count Not Decreasing

Problem: Stock display doesn't update after adding items to cart.
Cause: updateStockDisplay() not called or missing data-sku attributes.
Solution: Ensure all products have data-sku and data-stock attributes, call updateStockDisplay() after inventory changes.

Debugging Tips

  • Check browser console for JavaScript errors (F12)
  • Inspect localStorage: localStorage.getItem('artshop_cart')
  • Verify XAMPP Apache is running (green status in XAMPP Control Panel)
  • Clear localStorage to reset: localStorage.clear()
  • Test on actual devices, not just browser dev tools

Code Organization

  • Keep JavaScript functions modular and reusable
  • Use consistent naming conventions (camelCase for functions, kebab-case for CSS)

JavaScript Implementation

// Initialize search functionality document.addEventListener("DOMContentLoaded", function () { const searchForm = document.getElementById('search-form'); const searchInput = document.getElementById('search-input'); if (searchForm && searchInput) { searchForm.addEventListener('submit', function(e) { const query = searchInput.value.trim(); if (query) { // Form will submit naturally to Google search console.log('Searching for:', query); } else { e.preventDefault(); alert('Please enter a search term'); } }); // Make search input more visible searchInput.style.minWidth = '150px'; searchInput.style.padding = '5px 10px'; } });

How It Works

  • Action URL: Points to https://www.google.com/search
  • Method: GET request with query parameter "q"
  • Target: Opens results in new tab (_blank)
  • Validation: Required attribute ensures non-empty searches
  • User Experience: Seamless integration without leaving the site context

Key Features Implemented

1. Immediate Payment Capture

  • Intent: CAPTURE - Customer charged immediately upon approval
  • No authorization hold - funds transferred instantly
  • Ideal for digital goods and immediate fulfillment

2. Payment Method Vaulting

  • Stores customer PayPal account for future purchases
  • Vault ID saved in database with order record
  • Enables one-click checkout for returning customers
  • Configuration: store_in_vault: 'ON_SUCCESS'

3. Order Processing Flow

1. Customer clicks PayPal button
2. create-order.php creates PayPal order with cart data
3. Customer logs into PayPal and approves payment
4. capture-order.php captures payment immediately
5. Order saved to database with vault ID
6. Confirmation email sent to customer
7. Cart cleared, customer redirected to success page

4. Email Confirmation System

  • Trigger: Sent immediately after successful payment capture
  • Content: Order ID, transaction ID, items list, total amount
  • Format: HTML email with styled template
  • Delivery: PHP mail() function (upgradeable to SMTP)
