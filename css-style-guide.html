Usage Guidelines & Best Practices

Design Tokens (CSS Variables)

Why use CSS variables? Design tokens ensure consistency across your entire project. When you need to update a color or spacing value, you only change it in one place (the :root), and it updates everywhere automatically.

/* Good - Using CSS variables */ .header { background: var(--primary-color); padding: var(--spacing-lg); } /* Avoid - Hardcoded values */ .header { background: #333333; padding: 24px; }

Color Palette & Accessibility

Maintain brand consistency: Always use colors from the defined palette. Ensure proper contrast ratios for text readability (WCAG AA requires 4.5:1 for normal text, 3:1 for large text). Dark text (#333) on white backgrounds and white text on dark backgrounds meet these requirements.

  • Primary Dark (#333333): Headers, navigation, main buttons
  • Secondary Blue (#5383d3): Links, accents, secondary actions
  • Background (#ffffff): Page backgrounds, card surfaces
  • Text Colors: Use #000 for primary text, #666 for secondary text

Spacing & Layout Consistency

Follow the spacing scale: Using consistent spacing creates visual rhythm and improves the overall design. Never use arbitrary pixel values—always reference the spacing variables.

/* Spacing hierarchy */ xs (4px) - Tight spacing between related elements sm (8px) - Small gaps, inline elements md (16px) - Standard spacing for most components lg (24px) - Section spacing, card padding xl (32px) - Major section separation 2xl (48px) - Large section breaks, hero spacing /* Example usage */ .card { padding: var(--spacing-md); margin-bottom: var(--spacing-lg); } .section { padding: var(--spacing-xl) 0; }

Typography Hierarchy

Use proper heading structure: Maintain semantic HTML with correct heading hierarchy. This improves SEO and accessibility for screen readers.

  • H1 (32px): One per page - main page title
  • H2 (24px): Major section headings
  • H3 (20px): Subsection headings
  • H4 (18px): Minor headings, card titles
  • Body (16px): Standard paragraph text
  • Small (14px): Captions, metadata, secondary info
/* Typography Hierarchy CSS */ /* Define typography variables in :root */ :root { --font-family: "Manrope", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif; --font-size-h1: 32px; --font-size-h2: 24px; --font-size-h3: 20px; --font-size-h4: 18px; --font-size-base: 16px; --font-size-small: 14px; } /* Heading styles with semantic HTML */ h1 { font-size: var(--font-size-h1); font-weight: 700; line-height: 1.2; margin-bottom: var(--spacing-lg); color: var(--primary-color); } h2 { font-size: var(--font-size-h2); font-weight: 600; line-height: 1.3; margin-bottom: var(--spacing-md); color: var(--primary-color); } h3 { font-size: var(--font-size-h3); font-weight: 600; line-height: 1.4; margin-bottom: var(--spacing-md); color: var(--primary-color); } h4 { font-size: var(--font-size-h4); font-weight: 500; line-height: 1.4; margin-bottom: var(--spacing-sm); color: var(--primary-color); } /* Body text */ body { font-family: var(--font-family); font-size: var(--font-size-base); line-height: 1.6; color: var(--text-color); } p { margin-bottom: var(--spacing-md); } /* Small text utility */ .text-small { font-size: var(--font-size-small); color: #666; } /* HTML Example */ <h1>Main Page Title</h1> <h2>Major Section Heading</h2> <p>Standard body paragraph text with good readability.</p> <h3>Subsection Heading</h3> <h4>Minor Heading</h4> <p class="text-small">Small text for captions or metadata.</p>

Component Best Practices

Buttons

  • Use .btn-primary for main actions (Submit, Add to Cart)
  • Use .btn-secondary for secondary actions (Cancel, Learn More)
  • Always include hover states for visual feedback
  • Make buttons large enough for touch targets (minimum 44x44px)

  • Keep card heights consistent within the same grid

  • Include hover effects to indicate interactivity
  • Always use card-footer for pricing and actions

    Responsive Design: includes Mobile-first Approach, Testing on Devices, Touch Targets, Flexible Units.

    • Mobile-first approach: All grids collapse to single column below 768px
    • Test on actual devices, not just browser dev tools
    • Ensure touch targets are large enough (44x44px minimum)
    • Use flexible units (%, vh, vw) instead of fixed pixel widths
      • /* Responsive Design CSS */ /* Mobile-first Grid System */ .card-grid { display: grid; grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)); gap: var(--spacing-lg); } /* Breakpoint at 768px */ @media (max-width: 768px) { /* Collapse multi-column grids to single column */ .grid-2, .grid-3, .grid-4 { grid-template-columns: 1fr; } /* Reduce padding on mobile */ .style-guide-container { padding: var(--spacing-md); } /* Stack navigation vertically */ .nav-menu { flex-direction: column; gap: var(--spacing-sm); } /* Adjust font sizes for mobile */ .style-guide-header h1 { font-size: 32px; } /* Full-width cards on mobile */ .card-grid { grid-template-columns: 1fr; } } /* Flexible units instead of fixed pixels */ .container { width: 100%; max-width: 1200px; margin: 0 auto; padding: 0 5%; /* Percentage-based padding */ } /* Responsive images */ img { max-width: 100%; height: auto; } /* Touch-friendly buttons (minimum 44x44px) */ .btn { min-height: 44px; min-width: 44px; padding: 10px 20px; }

      Maintenance & Updates

      Keeping your design system current: When adding new components or updating existing ones, always check that they follow these guidelines and use the established design tokens.

      /* When creating new components, use existing tokens */ .new-component {display: flex; flex-direction: column; padding: var(--spacing-md); background: var(--background-color); border-radius: var(--border-radius-md); box-shadow: var(--shadow-sm); color: var(--text-color); font-family: var(--font-family); } /* Update tokens in :root to affect all components */ :root { --primary-color: #333333; /* Update here to change everywhere */ }
